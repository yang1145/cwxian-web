## 1. 文档概述

### 1.1 项目简介

创无限是一个面向小型创业企业与个人创业者的公益性互联网基础资源服务平台，官方域名为 **cwxian.com**。平台核心服务包括：

- **免费二级域名**：为符合条件的创业项目提供 `*.cwxian.com` 或自定义前缀的二级域名解析服务；
- **免费托管空间**：为项目提供基础网站/应用托管空间，降低创业初期的服务器成本；
- **创业资源链接**：聚合域名、空间、文档、社区等基础服务，帮助创业者快速搭建线上门面。

### 1.2 PRD 范围

本文档聚焦**创无限主站（cwxian.com）**的产品规划。主站是一个**纯前端展示型官网**，不承担域名注册、空间开通、用户账户、审核管理等后端业务功能。

平台整体由三个站点组成：

| 站点 | 域名 | 定位 | 与主站关系 |
|------|------|------|------------|
| **创无限主站** | cwxian.com | 品牌展示、规则说明、使用引导、流量入口 | 本文档规划范围 |
| **域名服务子站** | dns.cwxian.com | 二级域名的注册、解析、管理 | 主站通过 CTA 跳转引导 |
| **空间服务子站** | free-host.cwxian.com | 免费托管空间的开通、文件管理 | 主站通过 CTA 跳转引导 |

dns.cwxian.com 与 free-host.cwxian.com 仅保留最基础的功能，**不展示规则说明、使用教程、审核标准等内容**。因此，主站必须承载所有必要的信息：服务介绍、申请条件、使用规则、操作指引、常见问题、协议政策等，确保用户在跳转到子站之前已经充分了解所需信息。

本文档涵盖：

- 主站面向的目标用户与价值定位；
- 主站的信息架构、页面规划与核心功能；
- 各页面的内容要求、交互逻辑与引导跳转策略；
- 非功能层面的体验、安全、合规与可访问性要求；
- 关键成功指标与上线里程碑。

本文档**不涉及具体技术栈、框架选型、部署架构或代码实现细节**，也**不涉及 dns.cwxian.com 与 free-host.cwxian.com 的子站功能设计**，仅定义主站应“做什么”与“做到什么程度”。

## 2. 项目背景与目标

### 2.1 市场背景

对于早期创业项目而言，线上门面的建设成本（域名、服务器、备案、运维）往往是第一笔不可忽视的开支。许多小型创业团队在产品验证阶段面临以下问题：

- 资金有限，难以承担域名与服务器年付成本；
- 技术储备不足，不知如何完成域名解析与网站部署；
- 需要一个低成本、可快速上线的方案来验证商业假设；
- 希望获得一个稳定的二级域名作为早期品牌入口。

### 2.2 产品愿景

> 让每一个有价值的创业想法，都能以最低成本拥有一个专业的线上起点。

创无限希望通过提供免费的二级域名与托管空间，消除创业初期的基础设施门槛，让创业者将有限资源投入到产品与市场验证中。

### 2.3 服务架构说明

创无限平台由主站与两个子站协同构成：

- **主站（cwxian.com）**：承担品牌展示、规则说明、使用引导、流量汇聚的职责。由于子站仅保留最基础功能，主站需要预先向用户展示完整的服务规则、申请条件、操作流程与常见问题，避免用户在子站遇到困惑。
- **域名服务子站（dns.cwxian.com）**：仅提供域名注册、解析记录管理等最基础功能。用户从主站了解规则后，点击跳转至该子站完成注册。
- **空间服务子站（free-host.cwxian.com）**：仅提供空间开通、文件上传等最基础功能。用户从主站了解规则后，点击跳转至该子站完成开通。

### 2.4 官网目标

创无限主站是平台的**官方入口、信任背书与流量分发中心**，其核心目标包括：

1. **品牌传递**：清晰传达创无限的使命、服务内容与公益属性；
2. **信任建立**：通过案例展示、运营数据、合作伙伴与合规信息建立用户信任；
3. **引流跳转**：引导符合条件的用户跳转至 dns.cwxian.com 或 free-host.cwxian.com 完成注册/开通；
4. **规则承载**：在主站完整展示服务规则、申请条件、使用限制、审核标准，避免子站功能缺失导致用户迷茫；
5. **服务支持**：提供使用文档、常见问题与客服入口，降低用户使用门槛；
6. **社区连接**：展示入驻项目、开源贡献与志愿者计划，形成创业生态。

### 2.5 业务目标

| 目标层级 | 目标描述 | 衡量方式 |
|----------|----------|----------|
| 品牌目标 | 成为国内早期创业者首选的免费域名与托管服务平台 | 品牌词搜索量、自然流量占比 |
| 用户目标 | 上线 12 个月内通过子站服务 500+ 个入驻项目 | 子站审核通过并成功启用的项目数（由子站统计，主站定期同步展示） |
| 转化目标 | 官网至子站的跳转转化率达到 15% 以上 | 子站跳转点击数 / 官网访问 PV |
| 内容目标 | 帮助文档覆盖用户 80% 以上的常见问题 | 帮助中心页面浏览量、用户反馈 |
| 满意度目标 | 用户问题 24 小时内响应率达到 90% | 客服工单/邮件响应时效 |

## 3. 目标用户与用户画像

### 3.1 核心用户群体

创无限官网主要面向以下四类用户：

#### 3.1.1 早期创业团队（核心用户）

- **特征**：2-5 人小团队，处于 MVP 验证或产品冷启动阶段；
- **痛点**：预算有限，需要低成本上线官网、Demo 或着陆页；
- **诉求**：免费、稳定、易用的域名与托管服务；
- **决策因素**：申请门槛、审核速度、服务稳定性、品牌可信度。

#### 3.1.2 独立开发者 / 独立创客

- **特征**：个人开发者，拥有独立项目或 side project；
- **痛点**：不愿为实验性项目购买独立域名与服务器；
- **诉求**：快速获得一个可分享的二级域名，用于展示作品；
- **决策因素**：申请便捷性、是否支持自定义子域名、空间大小。

#### 3.1.3 高校创业 / 学生团队

- **特征**：大学生创业比赛团队、课程项目团队；
- **痛点**：缺乏持续资金来源，项目生命周期短；
- **诉求**：免费资源支持比赛展示与项目部署；
- **决策因素**：公益属性、申请流程简单、是否有教育场景案例。

#### 3.1.4 合作方与志愿者

- **特征**：云服务厂商、开源社区、技术志愿者、投资机构；
- **诉求**：了解平台运营模式、寻求合作机会或参与志愿贡献；
- **决策因素**：平台透明度、社区活跃度、合作回报。

## 4. 产品定位与价值主张

### 4.1 定位声明

创无限是**面向早期创业项目与小团队的免费互联网基础资源服务平台**，通过提供免费二级域名与托管空间，帮助创业者以零成本快速建立线上品牌与产品展示入口。

### 4.2 核心价值主张

- **零成本启动**：域名与托管空间完全免费，降低创业初期基础设施投入；
- **快速上线**：简化申请与配置流程，项目可在通过审核后快速启用；
- **稳定可靠**：平台承诺提供稳定的基础服务与基本技术支持；
- **公益属性**：不以盈利为首要目标，优先服务有真实创业需求的团队；
- **生态连接**：连接创业团队、志愿者、投资人与技术社区。

### 4.3 差异化优势

| 维度 | 创无限 | 传统云服务商 | 免费静态托管平台 |
|------|--------|--------------|------------------|
| 域名 | 提供免费二级域名 | 需付费购买 | 通常不提供自定义域名 |
| 空间 | 提供免费托管空间 | 需付费购买 | 空间有限或仅支持静态 |
| 目标用户 | 早期创业团队 | 全阶段企业与开发者 | 开发者与个人项目 |
| 审核机制 | 项目资质审核 | 付费即用 | 通常无需审核 |
| 社区支持 | 创业资源与社区连接 | 商业技术支持 | 社区驱动 |

#### 页面模块

| 模块 | 内容要求 |
|------|----------|

| 免费二级域名服务 | 服务内容、可选域名格式、解析支持、续期规则、禁用规则、子站跳转入口 |
| 免费托管空间服务 | 空间规格、支持的技术类型、上传方式、流量限制、安全限制、子站跳转入口 |
| 服务对比表 | 免费版服务规格一览，可预留未来高级版扩展说明 |

| CTA | 引导用户阅读开通指引后跳转至对应子站 |

#### 服务规格（产品定义）

**免费二级域名服务**（注册入口：dns.cwxian.com）：

- 域名格式：`*.cwxian.com`，前缀可自定义；
- 支持 DNS 解析记录管理（A、CNAME、TXT 等）；
- 每个项目可申请 1 个主域名，特殊情况可申请额外域名；
- 域名有效期与项目活跃度挂钩，长期不使用可能回收；
- 禁止注册涉及侵权、违法、敏感词汇的域名前缀。

**免费托管空间服务**（开通入口：free-host.cwxian.com）：

- 提供静态网站托管与有限动态脚本支持；
- 每个项目默认分配固定存储空间与月流量；
- 支持通过 Web 界面上传文件或 Git 同步；
- 禁止用于文件存储、下载站、挖矿、代理、博彩等违规用途。

#### 页面模块

#### 开通流程

1. **阅读规则**：用户在主站了解服务条件、限制与审核标准；
2. **条件自检**：用户对照自检清单确认项目符合要求；
3. **点击跳转**：用户点击对应按钮前往 dns.cwxian.com 或 free-host.cwxian.com；
4. **子站注册/开通**：用户在子站完成实际的注册或开通操作；
5. **等待审核**：子站完成审核后通知用户；
6. **上线运行**：用户根据帮助中心教程配置域名解析或上传网站文件。

| 按钮 | 目标地址 | 文案建议 |

| 开通托管空间 | https://free-host.cwxian.com | 我已了解规则，前往开通空间 |

## 9. 用户体验设计原则

### 9.1 设计方向

- **简洁现代**：页面布局清晰，留白充足，信息层次分明；
- **响应式**：适配桌面端、平板端与移动端访问；
- **快速加载**：首页关键内容优先加载，减少用户等待；
- **无障碍**：遵循基本的无障碍设计原则，支持键盘导航与屏幕阅读器；
- **品牌一致性**：色彩、字体、图标风格与创无限品牌调性保持一致。

### 9.2 关键交互原则

- 长页面使用目录锚点或分块展示，减少用户滚动负担；
- 表单实时校验，错误提示明确（如联系表单）；
- 操作成功/失败均给予清晰反馈；
- 关键路径（了解服务 → 开通指引 → 跳转子站）减少跳转次数；
- 子站跳转入口在关键页面全局可见；
- 帮助入口全局可见。

### 9.3 移动端体验

- 移动端导航采用汉堡菜单或底部导航；
- 表单字段在移动端优先使用原生输入控件；
- 案例卡片、服务卡片在移动端单列展示；
- 子站跳转按钮在移动端突出显示，便于点击。

## 10. 非功能性需求

### 10.1 性能要求

- 首页首屏加载时间不超过 3 秒（主流网络环境）；
- 页面交互响应延迟不超过 200ms；
- 核心页面（首页、服务页、开通指引页）支持预渲染或静态化，提升加载速度与 SEO 效果；
- 图片、字体等静态资源进行压缩与缓存优化；
- 尽量减少外部脚本依赖，避免阻塞首屏渲染。

### 10.2 安全要求

- 联系表单等前端表单需进行基础有效性校验与防垃圾信息过滤；
- 防止常见 Web 攻击（如 XSS、点击劫持等）；
- 子站跳转链接使用明确且可识别的域名，避免用户被误导至钓鱼网站；
- 用户协议、隐私政策等页面内容清晰展示，避免法律风险；
- 域名与空间服务的滥用监控与处置由子站负责，主站需展示相关规则与举报方式。

### 10.3 可用性要求

- 官网整体可用性目标不低于 99.9%；
- 关键路径（首页访问、服务页浏览、子站跳转）具备容错能力；
- 建立监控告警机制，及时发现并处理页面访问异常；
- 由于主站无后端，联系表单等可对接第三方服务或提供备用联系方式。

### 10.4 可访问性要求

- 所有图片提供替代文本（alt）；
- 表单字段关联正确的 label；
- 页面支持键盘操作；
- 颜色对比度符合 WCAG 2.1 AA 标准；
- 重要操作不仅依赖颜色区分状态。

### 10.5 SEO 与社交媒体

- 每个页面设置独立的标题、描述与关键词；
- 首页与核心页面支持结构化数据（Organization、WebSite 等）；
- 支持 Open Graph 与 Twitter Card，便于社交媒体分享；
- 生成并维护 XML 站点地图；
- 设置合理的 robots.txt 规则。

### 10.6 合规要求

- 官网底部展示 ICP 备案号（中国大陆运营）；
- 用户协议与隐私政策页面完整、可访问；
- 收集用户信息时明确告知用途并取得同意；
- 建立内容审核机制，及时处理违法违规内容；
- 遵守《网络安全法》《数据安全法》《个人信息保护法》等相关法规。
